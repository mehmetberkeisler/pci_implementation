What Is PCI?

The Scientific Question

Can we objectively measure the level of consciousness in a person who cannot communicate? Patients under anaesthesia, in coma, vegetative state, or minimally conscious state cannot report their subjective experience. The Perturbational Complexity Index (PCI) provides a theory-grounded, single-number measure that quantifies how complex the brain's response is to a direct electromagnetic perturbation, without requiring any behavioural cooperation.

The Idea

Consciousness requires the brain to be both integrated (parts talk to each other) and differentiated (each part does something unique). PCI captures both by perturbing the cortex with TMS (Transcranial Magnetic Stimulation) and measuring how complex the resulting spatiotemporal EEG pattern is using Lempel-Ziv compression.

What TMS-EEG Is

A TMS coil delivers a brief magnetic pulse to the cortex, triggering a cascade of neural activation. High-density EEG (32-256 channels) records the brain's response over ~350 ms. This combined TMS-EEG protocol captures how the perturbation propagates through cortical networks.

Clinical Significance

PCI correctly classified consciousness levels in 100% of healthy awake subjects and distinguished conscious from unconscious states across diverse conditions: NREM sleep, anaesthesia (midazolam, propofol, xenon), locked-in syndrome, vegetative state, and minimally conscious state (Casali et al. 2013).

Project Overview

What This Software Does

This project is a complete, end-to-end implementation of the PCI algorithm. Starting from raw BrainVision TMS-EEG recordings, it performs the full processing pipeline: artifact removal, filtering, epoching, source estimation, bootstrap statistics, Lempel-Ziv compression, and normalization, then presents results through an interactive Streamlit web interface with quality assessment and diagnostic visualizations.

Key Features

  • Automated trigger classification (TMS / Response / Annotation)
  • Istisna mode: exception handling for mislabeled TTL markers
  • CSD (Current Source Density) or sensor-level analysis
  • Correlated bootstrap with Global Maximum Statistics
  • Dead channel auto-exclusion
  • Temporal PCI evolution PCI(t)
  • Comprehensive quality assessment with physiological range checks

How the Algorithm Works — Step by Step

The flowchart below traces the complete processing pipeline from raw EEG data to the final PCI value. Each step includes a plain-language explanation of what it does and why it matters. Green FIXED tags mark issues that were identified and corrected during code review.

1Load & Parse Input
Input: BrainVision files (.vhdr + .vmrk + .eeg) — a standard format for TMS-EEG recordings.
What happens: The software reads the raw EEG recording using MNE-Python. It extracts all event markers (annotations) embedded in the file. These markers indicate when TMS pulses were delivered and when the subject pressed buttons. If hardware-level stimulus channels exist (Stim_XX), those events are recovered as well.
MNE Loading
mne.io.read_raw_brainvision()
Extracts events, event IDs, and metadata (sampling frequency, channel names, recording duration).
Stim Channel Recovery
mne.find_events(stim_channel="auto")
Recovers hardware-level Stim_XX events not present in annotations.
2Trigger Classification
What happens: Each event marker from the recording is automatically classified into a category based on its name and frequency. This is crucial because the software needs to identify which events correspond to TMS pulses (the "perturbation") as opposed to button presses, recording annotations, or other artifacts. The user selects from the classified triggers to specify which event should be used as the TMS timing reference.
For each event marker, classify_trigger(name, count) assigns a category:
TMS_LIKELY
Matches stimulation keywords ("S 1", "Stimulus/", "Stim_") AND count is in the typical 30-500 pulse range.
TMS_POSSIBLE
Has stimulation keywords but an unusual count (<30 or >500), suggesting a non-standard protocol.
RESPONSE
Matches response patterns ("Response", "button", "key", "R 1") — typically button presses.
ANNOTATION / OTHER
Recording boundaries, comments, or unclassified events.
2bIstisna Mode — Exception Handling improved
What happens: In some TMS-EEG setups, the hardware TTL pulses from the TMS stimulator are accidentally recorded as "Response" markers instead of "Stimulus" markers due to port wiring. The istisna (Arabic for "exception") mode detects this situation by analysing the temporal regularity of the response events. Real button presses are irregular; TMS pulses are periodic. If a response-labeled train shows clock-like regularity, the user is offered the option to reclassify it as TMS.
Only for RESPONSE markers. Detects mislabeled TTL pulses:
Periodicity Detection Algorithm
detect_stimulation_like_response_train()
1. Require at least 30 total events
2. Split into temporal blocks by gaps > 12 seconds
3. For each block with at least 10 events:
    a. Compute inter-stimulus intervals (ISI) in seconds
    b. Check median ISI is in [0.2, 10] seconds (typical TMS rates)
    c. Check coefficient of variation CV(ISI) ≤ 0.05 (5% jitter tolerance) was 0.02
4. If any block passes both criteria → is_stimulation_like = True
Is periodic AND user enables Istisna checkbox?
YES → Treat response markers as TMS triggers (optional per-block selection) NO → Blocked from computation unless Expert Override is enabled
3Preprocessing Pipeline
What happens: The raw EEG signal is cleaned and prepared for analysis. TMS pulses create massive electromagnetic artifacts that must be removed. The signal is then filtered to the relevant frequency band, downsampled to match the original paper's parameters, and cut into short segments (epochs) centred on each TMS pulse. Noisy epochs are rejected. Finally, the signal-to-noise ratio is computed as a data quality indicator.
3a. TMS Artifact Interpolation
Linear interpolation over [−2, +10] ms around each TMS pulse. Replaces the massive artifact with a smooth ramp connecting the boundary samples.
validate_window_overlap() warns if post-stimulus analysis starts before interpolation ends. new
3b. Bandpass Filter
0.1–45 Hz zero-phase FIR filter (MNE default). Removes slow drifts and high-frequency noise while preserving the cortical response band.
3c. Resample
Downsample to 362.5 Hz if original sampling rate > 400 Hz. Matches the temporal resolution used in Casali 2013 for cross-study comparability.
3d. Epoch & Reject
Cut [−500, +350] ms segments around each TMS pulse. Reject epochs where any channel exceeds 150 μV peak-to-peak (likely muscle/movement artifact).
3e. SNR Computation
SNR = mean|amplitude|25–300 ms ⁄ mean(σbaseline)
Sessions with SNR < 1.4 were excluded by Casali et al. (2013). This threshold ensures the TMS-evoked response is detectable above background noise.
4Source Estimation CSD ≠ inverse solution
What happens: Scalp EEG signals reflect the mixed activity of many cortical sources. This step transforms the raw channel data into estimates of cortical current density, improving spatial resolution. Our implementation offers two modes: (a) CSD (Current Source Density) via the surface Laplacian, which is a spatial filter that sharpens the topography, or (b) direct sensor-level analysis. The original paper used a full inverse-solution (3-sphere BERG model) with ~5000 cortical dipoles, which is a more sophisticated approach.
CSD Transform (default)
Surface Laplacian via mne.preprocessing.compute_current_source_density().
Produces 1 "source" per electrode channel (typically 32–256).
Sensor-Level (fallback)
Raw EEG channel voltages used directly as sources.
Simpler but more affected by volume conduction.
Important Caveat
The published PCI thresholds (0.31 / 0.44) were calibrated using full inverse-solution source reconstruction (~5000 dipoles), NOT CSD. Our CSD-based PCI values are not directly comparable to these reference ranges. Casarotto et al. (2016) demonstrated that sensor-level PCI retains discriminative power, but thresholds may need recalibration. documented
Output: source_data — 3D array of shape (n_sources × n_times × n_trials)
5Bootstrap Significance Matrix critical fix applied
What happens: This is the statistical heart of the algorithm. It determines which spatiotemporal points in the brain response are significantly different from pre-stimulus baseline activity. For each source, the trial-averaged response is z-scored against its baseline. A bootstrap procedure builds a null distribution by randomly resampling baseline data, then sets a global threshold. Any z-score exceeding this threshold is marked as significantly active, producing a binary matrix SS(x,t) of 0s and 1s.
5a. Trial-Average
evoked = mean(source_data, axis=trials)
Average across all accepted epochs to extract the reproducible evoked response, suppressing random noise. Output: (n_sources × n_times).
5b. Z-Score per Source
For each source: μ and σ from baseline [−500, −1] ms (ddof=1).
Dead channels (σ ≈ 0) are excluded entirely rather than floored. fixed
5c. Bootstrap Null Distribution — Global Maximum Statistics
Spatially-correlated resampling (same time indices across ALL sources simultaneously) critical fix
for b in range(500):
  idx = random_time_indices(n_post)       # ONE set of indices
  sampled = z_baseline_alive[:, idx]     # SAME indices for ALL sources
  null_maxima[b] = max(|sampled|)        # global max across sources & times
Why this matters: The previous code used independent random indices per source, which broke the natural spatial correlation structure between brain regions. This produced a less conservative (lower) threshold, admitting more false positives into SS. The corrected version preserves cross-source correlations, matching the Global Maximum Statistics framework (Nichols & Holmes, 2002) used in neuroimaging.
5d. Threshold
τ = 99th percentile of null maxima
(significance level α = 0.01)
5e. Binary Matrix
SS(x,t) = 1 if |z(x,t)| > τ, else 0
Excluded dead channels → all zeros automatically.
6Quality Gates
What happens: Before computing PCI, the algorithm checks whether the binary matrix contains enough information to produce a meaningful result. If the matrix is almost all zeros (no significant response) or almost all ones (noise-dominated), the entropy is too low and the Lempel-Ziv normalization becomes unstable. In these cases PCI is set to zero.
p1 = Σ SS / |SS|      H(L) = −p1 log2(p1) − (1 − p1) log2(1 − p1)
H < 0.08  OR  p1 < 0.01?
YES → PCI = 0 (insufficient signal) — STOP NO → Continue to compression
Same thresholds now used in both compute_pci and compute_pci_temporal. harmonised
7Optimal Ordination & Lempel-Ziv Compression
What happens: This is the core information-theoretic step. First, the rows (sources) of the binary matrix are sorted by their total activation count, placing the most active sources at the top. This "optimal ordination" minimises artefactual spatial complexity. Then the 2D matrix is flattened into a 1D binary string and compressed using the Lempel-Ziv 1976 algorithm. The number of distinct "words" found during parsing measures the sequence's algorithmic complexity — how much novel information it contains. A highly complex (differentiated + integrated) brain response produces many distinct words; a simple, repetitive pattern produces few.
7a. Source Sorting
Sort rows by descending total activation with deterministic tie-breaking:
_sort_sources(SS) uses primary key (total 1s) and secondary key (row content as binary number) to ensure permutation-invariant ordering. fixed
7b. Column-Major Flatten
Scan the matrix time-by-time (Casali convention):
flat = SS_sorted.flatten(order='F')
This scans: [all_sources_at_t0, all_sources_at_t1, ...] — concatenating the full spatial vector at each timestep.
7c. LZ76 Exhaustive Parsing
c = 1,   i = 1                        # first symbol is word #1
while i < len(sequence):
  find longest prefix s[i:i+k] that exists in history s[0:i]
  i += max_match + 1             # new word = match + 1 novel symbol
  c += 1
return c                           # = cL (spatial complexity count)
7d. Temporal Variant (PCIT)
The transposed matrix is also compressed: c_L_T = LZ76(SS_sorted.T.flatten('F')). This scans source-by-source (each source's full time series concatenated), capturing temporal complexity as complementary information to the spatial variant.
8Normalisation → PCI
What happens: The raw LZ complexity count depends on the matrix size and the fraction of 1s, making it hard to compare across recordings with different numbers of channels or time points. The normalisation divides by the theoretical asymptotic LZ complexity of a random binary sequence with the same entropy — the maximum complexity a random matrix of that size and density could achieve. This produces PCI, a dimensionless index between 0 and ~1 that is comparable across subjects, sessions, and recording setups.
PCI = cL × log2(L) ⁄ [ L × H(L) ]
Parameters
L = n_sources × n_post_times
H(L) = source entropy of SS
Normaliser = L × H ⁄ log2(L)
(the asymptotic cL of a random binary string with the same p1)

Corrections & Improvements Applied

Critical Fixes

  • Bootstrap resampling: Changed from independent per-source indices to correlated resampling (same time indices across all sources), preserving spatial correlations per Global Maximum Statistics (Nichols & Holmes, 2002).
  • Dead channel handling: Replaced sigma flooring with explicit exclusion of zero-variance sources, preventing inflated z-scores from flat electrodes.
  • Source sorting: Added deterministic tie-breaking using row content as binary number, ensuring permutation-invariant results.

Moderate Fixes

  • Window overlap validation: New warning when TMS artifact interpolation overlaps the post-stimulus analysis window.
  • Quality threshold harmonization: compute_pci_temporal now uses same min_entropy=0.08, min_p1=0.01 as compute_pci.
  • CSD documentation: Added prominent warnings that CSD ≠ inverse solution and published thresholds may not apply.
  • Istisna CV threshold: Raised from 0.02 to 0.05 to accommodate jitter from manual TMS triggering.
  • Configurable random seed: Bootstrap accepts seed= parameter.
