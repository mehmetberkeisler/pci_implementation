6. PCI ALGORITHM BEHAVIOR
════════════════════════════════════════════════════════════════════════════════

   The pci.py implementation applies these checks (lines 376-536):

   A) Creates epochs from trigger events
   B) Computes z-scores against baseline
   C) Bootstrap significance test (99th percentile threshold)
   D) Binary significance matrix SS(x,t)
   E) Quality gates:
      - Reject if H < 0.08 (insufficient entropy)
      - Reject if p₁ < 0.01 (insufficient activity)

   Why STIM/STIM2 Fail These Gates:
   ────────────────────────────────
   1. High baseline noise → z-scores are relative to noisy baseline
   2. All post-stimulus samples exceed threshold → SS becomes all 1s
   3. If p₁ ≈ 1.0 (all active), then H = -1×log₂(1) - 0×log₂(0) = 0
   4. H = 0 < 0.08 threshold → Quality gate rejects
   5. Returns PCI = 0.0 (invalid/rejected)

   This is CORRECT behavior - rejecting bad data, not a bug.
